# Project Guidelines

## Testing Requirements

- All new features MUST include corresponding tests.
- When modifying existing features, update or add related tests.
- Tests should cover both normal paths and edge cases.
- Test files should be placed alongside the code they test, using the `.test.ts` suffix (e.g., `code-review.test.ts` next to `code-review.ts`).

## Code Style

- Use English for code descriptions, comments, and documentation files.
- Use Chinese for planning documents.
